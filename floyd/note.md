## Floyd-Warshall 算法学习笔记

### 1. 问题描述 (Problem Description)

**目标：** 求解**全源最短路径 (All-Pairs Shortest Paths)** 问题。
即：在一个加权图中，计算图中任意两点 $(i, j)$ 之间的最短路径长度。

**输入条件：**
* 一个有向或无向图，用**邻接矩阵 (Adjacency Matrix)** 表示。
* 边权值可以为负数。
* **限制条件：** 图中不能包含“负权环”（Negative Cycle），否则最短路径可能不存在（无限小）。

---

### 2. 算法核心思想：动态规划 (Dynamic Programming)

Floyd 算法的本质是动态规划。它通过逐步增加可作为“中间跳点”的顶点集合，来不断更新最短路径。

#### 核心逻辑
假设图中有 $n$ 个节点，编号为 $0$ 到 $n-1$。
我们定义一个状态 $dist(k, i, j)$：表示从节点 $i$ 到节点 $j$，且中间节点只能从集合 $\{0, 1, 2, ..., k\}$ 中选择时的最短路径长度。

* **初始状态 ($k=-1$)：** $dist$ 矩阵即为原始邻接矩阵。
* **状态转移：** 当我们要考虑将节点 $k$ 加入中间点集合时，只有两种可能：
    1.  **不经过节点 $k$：** 最短路径依然是 $dist(k-1, i, j)$。
    2.  **经过节点 $k$：** 路径被拆分为 $i \to k$ 和 $k \to j$，总长度为 $dist(k-1, i, k) + dist(k-1, k, j)$。

**状态转移方程：**
$$dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j])$$



---

### 3. 分析过程 (Analysis Process)

#### A. 空间优化
虽然理论上需要三维数组 $dist[k][i][j]$，但在代码实现中，我们发现计算第 $k$ 层的状态只依赖于第 $k-1$ 层。因此，可以压缩空间，直接在**二维矩阵**上原地更新。

#### B. 时间复杂度
* **三层嵌套循环：** 外层循环控制中间节点 $k$，内层两层循环遍历所有起点 $i$ 和终点 $j$。
* **复杂度：** $O(n^3)$。对于节点数在 500 以内的图，该算法效率非常理想。

#### C. 代码逻辑拆解 (基于你提供的 Python 代码)

1.  **初始化：** `dist = copy.deepcopy(graph)`。将直接相连的边权值存入矩阵，不连通的设为 `INF`，对角线设为 0。
2.  **三重迭代：**
    * `for k in range(n)`: 决定当前允许经过哪个“新”节点。
    * `for i in range(n)`: 遍历每一行（起点）。
    * `for j in range(n)`: 遍历每一列（终点）。
3.  **松弛操作 (Relaxation)：** `if dist[i][k] + dist[k][j] < dist[i][j]:` 
    这行代码是算法的灵魂。它判断“曲线救国”（经过 $k$）是否比“直达”（当前的 $i \to j$）更近。

### 4. 示例走读 (Example Walkthrough)

以你代码中的矩阵为例：
`0 -> 2 (权值 3)`, `2 -> 3 (权值 1)`，初始时 `0 -> 3` 是 `INF`。

1.  **Iteration 0 (k=0):** 尝试经过节点 0，无明显更新。
2.  **Iteration 1 (k=1):** 尝试经过节点 1。
3.  **Iteration 2 (k=2):** * 发现 `dist[0][2] (3) + dist[2][3] (1) = 4`。
    * 因为 $4 < \infty$，所以 `dist[0][3]` 更新为 4。
4.  **Iteration 3 (k=3):** 尝试经过节点 3，继续优化其他路径。

### 5. 算法优缺点总结

| 优点 | 缺点 |
| :--- | :--- |
| 代码极其简洁（仅需三重循环）。 | 时间复杂度高 ($O(n^3)$)，不适合超大规模图。 |
| 能处理负权边。 | 不能处理包含负权环的图。 |
| 一次性算出所有点对的最短路径。 | |

> **笔记要点：** 记住核心公式 `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`。最外层循环一定是中间节点 `k`，这是初学者最容易写错的地方。
