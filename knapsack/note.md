## 0-1 背包问题学习笔记

### 1. 问题描述
给定 $n$ 个物品，每个物品有两个属性：重量 $w_i$ 和价值 $v_i$。现有一个容量为 $W$ 的背包。
* **规则**：每个物品只能选择 **放 (1)** 或 **不放 (0)**，不能切割。
* **目标**：在不超过背包容量的前提下，如何选择物品使得背包内物品的总价值最大。

---

### 2. 核心分析

#### A. 状态定义
我们需要一个变量来表示当前问题的规模。
* **$i$**：当前考虑到第几个物品。
* **$j$**：当前背包剩余的容量。
* **$dp[i][j]$**：表示前 $i$ 个物品放入容量为 $j$ 的背包中所能获得的最大价值。

#### B. 决策与状态转移
对于第 $i$ 个物品，我们只有两个选择：
1.  **不放入背包**：价值等于前 $i-1$ 个物品放入容量为 $j$ 的背包的价值。
    * 公式：$dp[i][j] = dp[i-1][j]$
2.  **放入背包**（前提是当前容量 $j \ge weights[i-1]$）：价值等于前 $i-1$ 个物品放入容量为 $j - weights[i-1]$ 的背包价值，加上当前物品的价值 $values[i-1]$。
    * 公式：$dp[i][j] = dp[i-1][j - weights[i-1]] + values[i-1]$

**综合状态转移方程：**
$$dp[i][j] = \max(dp[i-1][j], \quad dp[i-1][j - weights[i-1]] + values[i-1])$$



---

### 3. 解法演进分析

#### 方法一：递归 + 备忘录 (Top-down)
* **原理**：从最终状态出发，利用递归拆解为子问题。通过 `memo` 字典记录已计算过的状态 $(index, capacity)$，避免重复计算。
* **优点**：逻辑直观，仅计算必要的状态（如果某些容量状态永远达不到，则不会计算）。
* **缺点**：存在递归栈深度限制，在大数据量下效率略低于迭代。

#### 方法二：二维 DP (Bottom-up)
* **原理**：从小规模问题开始填表。
* **效率**：时间复杂度 $O(n \times W)$，空间复杂度 $O(n \times W)$。
* **特点**：稳定，逻辑清晰，适合理解 DP 的空间布局。

#### 方法三：一维 DP (Space Optimized)
* **优化思路**：观察发现 $dp[i][j]$ 仅依赖于上一行 $dp[i-1]$ 的数据。因此可以用一行数组不断更新。
* **核心关键**：**内层循环必须逆序**。
    * **原因**：正序遍历会导致 $dp[j - w]$ 在当前物品处理时被提前更新为“已放入当前物品”的状态，从而导致物品被重复计算（变成了完全背包问题）。逆序则保证了 $dp[j - w]$ 依然是“上一轮物品”的结果。
* **空间复杂度**：降至 $O(W)$。

---

### 4. 性能总结 (基于测试数据)

| 方法 | 时间效率 | 空间效率 | 结论 |
| :--- | :--- | :--- | :--- |
| **递归+备忘录** | 较慢 (57ms) | 较高 (字典存储) | 适合理解逻辑，处理稀疏状态。 |
| **二维 DP** | 中等 (20ms) | 较低 ($O(n \times W)$) | 最经典的教学写法，最稳定。 |
| **一维 DP** | **最快 (13ms)** | **最高 ($O(W)$)** | **工程实践中最推荐的写法。** |
